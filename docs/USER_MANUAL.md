## 1. 安装与启动

### 安装包

SuitAgent 提供两种安装方式，选择其一即可：

| 安装包 | 文件名 | 说明 |
|--------|--------|------|
| **MSI 安装包** | `SuitAgent_0.1.0_x64_en-US.msi` | 标准 Windows 安装向导，推荐使用 |
| **NSIS 安装包** | `SuitAgent_0.1.0_x64-setup.exe` | 可执行安装程序，功能相同 |

### 安装步骤

1. 双击安装包，按向导提示完成安装
2. 安装程序会自动部署以下组件：
   - SuitAgent 桌面主程序
   - 后端 API 服务（`suitagent-api.exe`）
   - 7 个预设工作流模板
3. 安装完成后，从开始菜单或桌面快捷方式启动 SuitAgent

### 前置依赖

- **Claude Code CLI**：SuitAgent 依赖 Claude Code 执行 AI 分析任务。请确保已安装并登录：
  ```bash
  # 安装 Claude Code（如尚未安装）
  npm install -g @anthropic-ai/claude-code

  # 登录
  claude login
  ```

### 启动流程

启动 SuitAgent 后，程序会自动：
1. 在后台启动 API 服务（端口 8000）
2. 等待服务就绪（通常 2-3 秒）
3. 打开主界面窗口（1280×800）

## 4. 工作流系统

工作流是 SuitAgent 的核心——它定义了一系列 AI Agent 的执行顺序，让多个分析步骤自动串联完成。

### 4.1 什么是工作流

一个工作流由以下部分组成：

- **名称**：工作流的标题
- **描述**：工作流的用途说明
- **步骤列表**：按顺序排列的 Agent 执行步骤，每个步骤指定一个 Agent 和操作标签
- **触发关键词**：用于场景自动识别的关键词列表
- **预期输出**：工作流完成后应产出的文档清单

### 4.2 工作流编辑器

点击顶部导航栏的 **工作流** 进入编辑器。界面分为三栏：

```
┌──────────────┬──────────────────────────────────┬────────────┐
│   工作流列表   │         步骤编辑区                │  Agent 面板 │
│              │                                  │            │
│  系统预设     │  工作流名称                       │  输入层     │
│  · 被告应诉   │  工作流描述                       │  · 文档分析  │
│  · 原告起诉   │                                  │  · 证据分析  │
│  · ...       │  ⠿ 1  doc-analyzer  分析起诉状    │            │
│              │  ⠿ 2  researcher    法律检索       │  分析层     │
│  我的工作流   │  ⠿ 3  writer       起草答辩状     │  · 争议识别  │
│  · 自定义流程  │                                  │  · 法律研究  │
│              │  触发关键词: ...                   │  · ...     │
│              │                                  │            │
│ [+ 新建工作流] │  [保存更改]  [删除]               │  支持层     │
└──────────────┴──────────────────────────────────┴────────────┘
```

**左栏**：工作流列表，分为「系统预设」和「我的工作流」两组
**中栏**：选中工作流的详细编辑区
**右栏**：可用 Agent 面板，按四层架构分组

## 5. 自定义工作流

### 5.1 从预设复制

最快的方式是基于预设工作流创建自定义版本：

1. 在左栏点击一个**系统预设**工作流（如「被告应诉」）
2. 点击 **复制为自定义** 按钮
3. 系统创建一个名为「被告应诉 (副本)」的可编辑副本
4. 现在你可以自由修改这个副本

### 5.2 从零新建

1. 点击左栏底部的 **+ 新建工作流**
2. 系统创建一个空白工作流
3. 在中栏编辑名称和描述
4. 从右栏添加 Agent 步骤（见下文）

### 5.3 添加步骤

在右栏的 **可用 Agent** 面板中，点击任意 Agent 名称，即可在步骤列表末尾添加一个新步骤。

例如，要构建一个简单的案件分析流程：
1. 点击「文档分析」→ 添加步骤 1
2. 点击「争议识别」→ 添加步骤 2
3. 点击「法律研究」→ 添加步骤 3
4. 点击「摘要生成」→ 添加步骤 4

### 5.4 调整步骤顺序

拖拽步骤左侧的 **⠿** 手柄，上下移动到目标位置。松开后步骤编号自动更新。

### 5.5 编辑步骤标签

每个步骤默认使用 Agent 名称作为标签。点击标签文字可以修改，例如将「doc-analyzer」的标签改为「分析庭审笔录」，使流程更具可读性。

### 5.6 移除步骤

点击步骤右侧的 **×** 按钮即可移除。剩余步骤自动重新编号。

### 5.7 编辑触发关键词

在步骤列表下方的 **触发关键词** 输入框中，用逗号分隔填写关键词：

```
收到起诉状, 需要应诉, 准备答辩
```

这些关键词用于在案件工作台中快速匹配和推荐工作流。

### 5.8 保存与删除

- 编辑后，**保存更改** 按钮会自动出现，点击保存
- 点击 **删除** 可以删除自定义工作流（系统预设不可删除）

### 5.9 自定义工作流示例

#### 示例 1：快速案情摘要

适用场景：客户初次咨询，需要快速了解案件概况。

| 步骤 | Agent | 标签 |
|------|-------|------|
| 1 | doc-analyzer | 分析客户材料 |
| 2 | issue-identifier | 识别核心争议 |
| 3 | summarizer | 生成案情摘要 |

触发关键词：`客户咨询, 快速分析, 案情摘要`

#### 示例 2：上诉可行性评估

适用场景：收到一审判决后，评估上诉必要性。

| 步骤 | Agent | 标签 |
|------|-------|------|
| 1 | doc-analyzer | 解析判决书 |
| 2 | issue-identifier | 提取争议焦点 |
| 3 | researcher | 检索上诉判例 |
| 4 | strategist | 上诉可行性分析 |
| 5 | summarizer | 生成评估报告 |

触发关键词：`收到判决, 上诉分析, 判决评估`

#### 示例 3：证据补强方案

适用场景：发现证据链存在缺口，需要制定补充方案。

| 步骤 | Agent | 标签 |
|------|-------|------|
| 1 | evidence-analyzer | 现有证据评估 |
| 2 | researcher | 举证责任研究 |
| 3 | strategist | 补强策略制定 |
| 4 | writer | 补充证据清单 |

触发关键词：`证据不足, 补充证据, 证据补强`

## 6. 案件工作台

从案件列表点击案件进入工作台，界面分为三列：

### 6.1 文档中心（左列）

**上传文档**

1. 选择文档分类：
   - `04 - 📤 客户提供`
   - `05 - 📎 证据材料`
   - `07 - 📥 对方提交`
   - `08 - 🏛️ 法院送达`
   - `09 - 🎯 庭审笔录`
2. 点击文件选择框，浏览本地文件
3. 点击 **上传并提取**

**查看文档**

上传后，文档卡片会出现在列表中。点击卡片可在下方的「提取预览」区域查看自动提取的文本内容。

### 6.2 产物与审核（中列）

工作流执行后生成的分析产物（答辩状、研究报告等）在此管理。

**审核产物**

1. 点击顶部的产物标签切换查看
2. 阅读产物内容
3. 在审核区域填写审核意见
4. 点击 **通过** 或 **驳回**

**导出产物**

- 点击 **导出 DOCX** 导出为 Word 文档
- 点击 **导出 Markdown** 导出为 Markdown 文件
- 导出成功后显示文件保存路径

**审核历史**

产物下方显示所有审核记录，包括审核人、审核结果、审核意见和时间。

### 6.3 任务与运行（右列）

**执行工作流**

1. 在「执行工作流」区域的下拉框中选择一个工作流
2. 点击 **执行**
3. 系统自动将案件中所有已上传文档作为输入
4. 每个工作流步骤生成一个任务，按顺序执行

**任务列表**

执行后，任务列表显示所有步骤的执行状态：

| 状态 | 颜色 | 说明 |
|------|------|------|
| `pending` | 黄色 | 等待执行 |
| `running` | 蓝色 | 正在执行 |
| `completed` | 绿色 | 执行完成 |
| `failed` | 红色 | 执行失败 |
| `canceled` | 灰色 | 已取消 |

**任务操作**

- 失败的任务：点击 **重试任务** 重新执行
- 运行中的任务：点击 **取消任务** 停止执行

**运行详情**

点击 **展开面板** 查看选中任务的详细信息：
- Workflow Run ID
- External Task ID
- 执行次数
- 执行日志

**案件时间线**

页面底部按时间倒序展示所有案件事件记录。

## 7. 系统设置

点击顶部导航栏的 **设置** 进入配置页面。

### 7.1 Claude Code 配置

| 设置项 | 说明 |
|--------|------|
| **CLI 路径** | Claude Code 可执行文件路径。留空则使用系统 PATH 中的 `claude` 命令 |

| 设置项 | 说明 |
|--------|------|
| **案件存储目录** | 案件文件的保存位置。留空则使用项目默认的 `output/` 目录 |

### 7.3 偏好设置

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| **审核提醒** | 开启 | 产物生成后自动提醒进行审核 |
| **默认导出 Word** | 开启 | 导出产物时默认使用 `.docx` 格式 |
| **显示执行日志** | 关闭 | 在任务面板中显示详细的执行日志 |

文本输入框在停止输入 0.5 秒后自动保存，开关切换立即生效。

## 9. Agent 一览

SuitAgent 的 10 个 AI Agent 按四层架构组织，各司其职：

### 输入层 — 文档数据采集

| Agent | 功能 | 典型任务 |
|-------|------|----------|
| **doc-analyzer** (文档分析) | 解析各类法律文档，提取结构化信息 | 分析起诉状、解析合同、提取庭审笔录要点 |
| **evidence-analyzer** (证据分析) | 深度分析证据材料，评估证明力 | 证据三性质证（真实性、合法性、关联性）、证据链分析 |

### 分析层 — 智能分析与研究

| Agent | 功能 | 典型任务 |
|-------|------|----------|
| **issue-identifier** (争议识别) | 识别和归类核心争议焦点 | 提取争议点、分析法律关系、评估争议优先级 |
| **researcher** (法律研究) | 法条解读、判例检索、法律适用研究 | 查找相关法条、检索类案判例、分析法律适用路径 |
| **strategist** (诉讼策略) | SWOT 分析、策略制定、风险评估 | 制定应诉/起诉策略、评估胜诉可能性、资源配置建议 |

### 输出层 — 文书生成与报告

| Agent | 功能 | 典型任务 |
|-------|------|----------|
| **writer** (法律文书) | 起草各类法律文书 | 起诉状、答辩状、代理词、质证意见书、委托合同 |
| **reporter** (报告整合) | 整合所有分析成果，生成综合报告 | 综合案件报告、阶段性报告、起诉材料包 |
| **summarizer** (摘要生成) | 生成各类摘要和简报 | 案件摘要、庭审摘要、风险摘要、客户汇报 |

### 支持层 — 质量保证与辅助

| Agent | 功能 | 典型任务 |
|-------|------|----------|
| **scheduler** (日程管理) | 期限管理、工时统计 | 计算诉讼时效、管理开庭日期、记录工时 |
| **reviewer** (质量审查) | 智能审查器，质量把关专家 | 审查文书质量、检查法律引用准确性、跨 Agent 一致性检查 |

## 常见问题

**Q: 工作流执行失败怎么办？**
A: 在任务列表中点击失败任务的「重试任务」按钮。如果反复失败，检查设置中的 Claude Code CLI 路径是否正确，以及 Claude Code 是否已登录。

**Q: 可以同时运行多个工作流吗？**
A: 当前版本按顺序执行。建议等待一个工作流完成后再启动下一个。

**Q: 自定义工作流会丢失吗？**
A: 不会。自定义工作流保存在本地 `workflows/custom/` 目录下的 JSON 文件中，不会因为软件更新而丢失。

**Q: 系统预设工作流可以修改吗？**
A: 不能直接修改预设工作流。请点击「复制为自定义」创建副本后再编辑。

**Q: 如何更换案件文件的存储位置？**
A: 进入「设置」页面，在「案件存储目录」中填写新路径即可。
