## 📁 核心文件结构

SuitAgent案件模板采用三层文件结构，支持AI快速加载和人工详细查阅的分离设计。

### 1. 根目录文件（AI优先）

**`[案件编号]案件信息.md`**

- **位置**: `output/[案件编号]/[案件编号]案件信息.md`
- **用途**: 案件管理看板，所有Agent的首要上下文来源
- **内容**: 案件基本信息、当事人信息、争议焦点、任务清单、期限管理等
- **特点**: 轻量级，AI可快速加载和解析

### 2. 日程管理目录文件（人工查阅）

**`00 - 📅 日程管理/[案件编号]工时记录.md`**

- **位置**: `output/[案件编号]/00 - 📅 日程管理/[案件编号]工时记录.md`
- **用途**: 详细工时记录与费用核算
- **内容**: 月度工时统计、文件清单、案件进展摘要、费用汇总等
- **特点**: 详细完整，支持审计查阅，AI不优先加载

**`00 - 📅 日程管理/[案件编号].yaml`**

- **位置**: `output/[案件编号]/00 - 📅 日程管理/[案件编号].yaml`
- **用途**: 结构化期限管理，机器可读数据
- **内容**: 案件基本信息、法定期限数据、任务清单、费用预算等
- **特点**: Scheduler Agent专用数据源

## 🔄 文件职责分工

| 文件                  | 主要使用者          | 更新频率   | 职责                       |
| --------------------- | ------------------- | ---------- | -------------------------- |
| **案件信息.md** | 所有Agent、承办律师 | 每次工作后 | 案件全局概览、上下文来源   |
| **工时记录.md** | 承办律师、财务      | 每天/每周  | 详细工时记录、费用核算     |
| **yaml**        | Scheduler Agent     | 实时       | 结构化期限管理、自动化提醒 |

## 📊 Agent分层架构与目录映射

SuitAgent采用四层架构设计，将10个Agent按职能分为4个层级：

### 📥 输入层 (Input Layer) - 文档数据采集

#### DocAnalyzer

- **主要输出**: `02 - 📄 案件分析`
- **次要输出**: `04 - 📤 客户提供`、`07 - 📥 对方提交`、`08 - 🏛️ 法院送达`、`09 - 🎯 庭审笔录`
- **功能**: 分析各类法律文档，提取结构化信息

#### EvidenceAnalyzer

- **主要输出**: `05 - 📎 证据材料`
- **次要输出**: `07 - 📥 对方提交`
- **功能**: 深度分析证据材料，评估证明力

### 🔍 分析层 (Analysis Layer) - 智能分析与研究

#### IssueIdentifier

- **主要输出**: `03 - 🔍 法律研究`
- **功能**: 识别和归类争议焦点

#### Researcher

- **主要输出**: `03 - 🔍 法律研究`
- **次要输出**: `11 - 📚 参考文件`
- **功能**: 法条解读、判例检索、法律适用研究

#### Strategist

- **主要输出**: `02 - 📄 案件分析`
- **功能**: SWOT分析、策略制定、风险评估

### 📝 输出层 (Output Layer) - 文书生成与报告

#### Writer

- **主要输出**: `06 - 📝 法律文书`
- **次要输出**: `01 - 🤝 委托材料`
- **功能**: 起草各类法律文书

#### Reporter

- **主要输出**: `10 - 📊 综合报告`
- **功能**: 整合所有内容，生成综合报告

#### Summarizer

- **主要输出**: `10 - 📊 综合报告`
- **次要输出**: `09 - 🎯 庭审笔录`
- **功能**: 生成各类摘要和简报

### ⚙️ 支持层 (Support Layer) - 质量保证与辅助

#### Scheduler

- **主要输出**: `00 - 📅 日程管理`
- **次要输出**: `08 - 🏛️ 法院送达`
- **功能**: 期限管理、工时统计

#### Reviewer

- **主要输出**: 无（不直接输出文件）
- **功能**: 智能审查器，质量把关专家

## 📋 完整目录结构（12层架构）

案件目录采用12层标准化结构，Agent输出遵循以下映射关系：

```
00 - 📅 日程管理        → Scheduler（期限管理、工时统计）
01 - 🤝 委托材料        → Writer（委托合同、授权书）
02 - 📄 案件分析        → DocAnalyzer, Strategist（案件分析、策略）
03 - 🔍 法律研究        → IssueIdentifier, Researcher（争议焦点、法条检索）
04 - 📤 客户提供        → DocAnalyzer（客户材料）
05 - 📎 证据材料        → EvidenceAnalyzer（证据分析、质证）
06 - 📝 法律文书        → Writer（起诉状、答辩状等）
07 - 📥 对方提交        → DocAnalyzer, EvidenceAnalyzer
08 - 🏛️ 法院送达        → DocAnalyzer, Scheduler（传票、判决书）
09 - 🎯 庭审笔录        → DocAnalyzer, Summarizer（庭审记录）
10 - 📊 综合报告        → Summarizer, Reporter（案件摘要、报告）
11 - 📚 参考文件        → Researcher（参考案例、法条）
```

## 💡 使用说明

### AI处理阶段

- **优先读取**: 根目录的 `案件信息.md`获取案件概览
- **Agent输出**: 按照目录映射规范输出到对应目录
- **文件同步**: 关键进展从详细记录同步到概览文件，保持信息一致性

### 人工记录阶段

- **工时记录**: 在 `工时记录.md`中记录详细工作内容和时长
- **期限管理**: Scheduler Agent自动维护 `yaml`文件中的期限数据
- **目录定位**: 通过标准化目录结构快速定位各类文档
