# Agent输出标准规范

**版本**: 1.2
**最后更新**: 2026-01-01
**说明**: 定义所有Agent输出文件的命名规范、格式标准和质量要求（注：Agent目录映射关系详见 AgentMapping.md）

## 🎯 文档职责说明

本规范专注于**输出文件本身的标准**：
- ✅ 文件命名规范（日期前缀、格式）
- ✅ 文件内容格式（Markdown结构、模板）
- ✅ 质量要求和检查清单

**Agent目录映射关系**详见 [`AgentMapping.md`](./AgentMapping.md)

## 🚨 核心输出原则

### 文件格式要求
- **必须输出.md（Markdown）格式文件**
- **不要输出.json格式文件**
- 所有Agent输出均应使用Markdown格式进行结构化展示

### 文件命名规范

#### 🚨 日期前缀强制要求
**所有Agent输出文件名必须以日期前缀开头，格式为：YYMMDD（空格）文件名**

- **日期格式**: YYMMDD（年月日各两位）
- **分隔符**: 日期后必须加一个空格
- **示例**: 2026年1月1日 → `260101 `
- **获取方式**: 使用 `date "+%y%m%d"` 获取当前日期

#### 通用命名规则
- **日期前缀**: 必须以 `YYMMDD ` 开头
- **使用中文描述**: 文件名应使用简洁明了的中文描述
- **避免特殊字符**: 文件名中避免使用 `/ \ : * ? " < > |` 等特殊字符
- **统一后缀**: 所有文件统一使用 `.md` 后缀

#### 📂 文件类型分类

**一次性输出文件**（需日期前缀）:
- 分析报告、法律文书、摘要等一次性生成的文档
- 每次生成为新版本，保留历史版本

**持续维护文件**（无需日期前缀）:
- 工时记录、期限管理等长期更新的文件
- 单一文件持续更新，不保留历史版本

| Agent | 文件类型 | 标准输出文件名 | 说明 |
|-------|---------|----------------|------|
| **DocAnalyzer** | 一次性 | `YYMMDD [文件名]解析结果.md` | 文档分析结果 |

| **Writer** | 一次性 | `YYMMDD [文书类型].md` | 各类法律文书 |
| **Summarizer** | 一次性 | `YYMMDD [摘要类型]摘要.md` | 各类摘要报告 |
| **Reporter** | 一次性 | `YYMMDD [报告类型]报告.md` | 综合分析报告 |

## 📝 输出格式标准

### Markdown结构规范

#### 1. 标题层级
- **一级标题**: `# ` 用于文档主标题
- **二级标题**: `## ` 用于主要章节
- **三级标题**: `### ` 用于子章节
- **四级标题**: `#### ` 用于详细条目

#### 2. 内容结构
每个Agent输出应包含以下标准结构：

```markdown
# [文档主标题]

## 核心要点/关键信息
[总结性内容]

## 详细内容
[具体分析内容]

## [Agent特定的章节]
[根据Agent类型定制的专业内容]

## 总结/建议
[结论性内容]
```

#### 3. 格式规范
- **列表**: 使用 `-` 或 `*` 创建无序列表
- **强调**: 使用 `**粗体**` 突出重点
- **引用**: 使用 `> 引用内容` 标注重要引用
- **代码**: 使用 `` `代码` `` 标注技术术语或函数名
- **分隔线**: 使用 `---` 分隔重要章节

## 🔄 质量要求

### 内容质量标准
1. **准确性**: 确保所有信息准确无误
2. **完整性**: 涵盖所有必要的分析要素
3. **逻辑性**: 内容结构清晰，逻辑严密
4. **专业性**: 使用准确的法律术语

### 格式质量标准
1. **一致性**: 所有Agent遵循相同的格式规范
2. **可读性**: 结构清晰，易于阅读和理解
3. **标准化**: 统一使用Markdown格式
4. **完整性**: 包含所有必要的章节和内容
