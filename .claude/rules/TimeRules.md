# 时间管理强制规范

**版本**: 1.0
**最后更新**: 2026-01-01
**说明**: 定义SuitAgent系统的完整时间管理规范，确保所有时间戳的准确性和一致性

## 🚨 核心原则

**所有时间相关的信息必须基于系统本机时间，严禁使用虚构或估算的时间戳**

## 📋 时间获取强制要求

### 必需步骤

#### 1. 获取本机时间

**在更新时间戳前必须执行**

```bash
date "+%Y-%m-%d %H:%M:%S"  # 获取当前系统时间
date "+%Y-%m-%d"           # 获取当前日期
```

#### 2. 验证时间准确性

- 确认获取的时间与当前实际时间一致
- 检查时区设置是否正确（应为本地时区，如CST）
- 验证日期格式标准化

#### 3. 时间应用规则

- **文档创建时间**：必须使用当前系统时间
- **文件更新时间**：必须使用实际操作时间
- **立案时间**：基于当前日期推算合理的诉讼时间线
- **期限计算**：从当前日期开始计算剩余天数
- **预计结案时间**：基于诉讼程序的合理时间预估

## ⚡ 时间修正强制流程

**触发条件**：当发现时间戳错误时

### 修正步骤

1. **立即停止**

   - 停止相关操作
2. **重新获取系统当前时间**

   ```bash
   current_time=$(date "+%Y-%m-%d %H:%M:%S")
   current_date=$(date "+%Y-%m-%d")
   ```
3. **分析需要修正的时间点**

   - 立案日期（如已立案则使用实际立案日期，如未立案则用当前日期）
   - 文件创建时间（统一使用当前日期）
   - 诉讼时效起算点（基于争议发生日期）
   - 预计时间节点（基于合理诉讼周期）
4. **批量修正所有时间相关字段**

   - 案件信息中的所有日期字段
   - 时间线中的事件日期
   - 任务清单中的截止日期
   - 期限管理中的起止日期
   - 更新历史中的时间戳
5. **验证修正结果的时间逻辑性**
6. **记录修正日志在更新历史中**

## ✅ 时间一致性检查清单

### 时间逻辑一致性

- [ ] 所有时间戳不晚于当前系统时间
- [ ] 时间线顺序逻辑合理（过去→现在→未来）
- [ ] 诉讼程序时间间隔符合实际（如立案到开庭需合理时间）
- [ ] 剩余天数计算准确

### 时间格式标准化

- [ ] 日期格式统一：`YYYY-MM-DD`
- [ ] 时间格式统一：`YYYY-MM-DD HH:MM:SS`
- [ ] 剩余天数标记：`[X天]`
- [ ] 预计时间标记：`[预计]`或 `[预计时间]`

### 文档时间同步

- [ ] 案件信息文件与案件状态文件时间一致
- [ ] 输出文档时间与实际操作时间一致
- [ ] 更新历史时间戳反映实际修改时间

## 🚫 违规处理机制

### 禁止行为

- ❌ 使用虚构或估算的时间戳
- ❌ 忽略时间逻辑检查
- ❌ 混用不同时间格式
- ❌ 使用未来时间作为"已完成"事件的时间
- ❌ 不验证系统时间准确性直接使用

### 纠正措施

- 立即停止相关操作
- 重新获取系统当前时间
- 全面检查所有时间相关字段
- 批量修正错误时间戳
- 验证修正结果的时间逻辑性
- 记录修正日志在更新历史中
