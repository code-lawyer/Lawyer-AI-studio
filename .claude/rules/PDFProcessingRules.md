# PDF处理规范

**版本**: v1.0
**适用Agent**: DocAnalyzer, EvidenceAnalyzer, 及其他需要处理PDF的Agent

## 处理优先级规则

当处理PDF/图片文档时，必须按照以下优先级顺序进行：

### 优先级1：MCP服务（首先尝试）

**检查MCP配置**：
```bash
# 简化的检查命令
for dir in . .. ../.. ../../.. ../../../..; do
    if [ -f "$dir/.claude/mcp.json" ]; then
        if grep -q '"mineru"' "$dir/.claude/mcp.json" 2>/dev/null; then
            echo "✅ 找到MinerU MCP配置"
            mcp_configured=true
            break
        fi
    fi
done
```

**调用MCP工具**：
```python
mineru.parse_documents("/path/to/document.pdf")
```

**分支决策**：
- ✅ **配置存在且调用成功** → 使用MCP返回结果
- ❌ **配置不存在或调用失败** → 跳转到优先级2

### 优先级2：PDF技能（备用方案）

**使用条件**：
- MCP未配置
- MCP配置不正确
- MCP调用失败

**处理流程**：
```
1. 声明回退：
   "MinerU MCP不可用，将使用PDF技能进行OCR。"

2. 调用pdf技能提取文档内容

3. 接收识别结果
```

### 优先级3：命令行工具（最后回退）

仅在PDF技能也失败时使用：
- tesseract（OCR）
- pdftotext（PDF文本提取）

## 强制输出规范

### ✅ 必须遵循
- 只生成一个同名Markdown文件
- 只包含结构化信息
- 文件名：`{原文件名}.md`

### ❌ 禁止生成
- JSON文件
- 原始OCR文本文件
- 图片目录
- 布局信息文件
- 多个版本文件
- 质量评估报告

### 标准输出格式

```markdown
# [文件名] 解析结果

## 关键信息提取
- 字段1: 值1
- 字段2: 值2
- ...

## 处理信息
- OCR工具：mineru MCP / pdf skill
- 处理时间：YYYY-MM-DD HH:MM
```


## 信息提取标准

### 必提取字段

1. **案件编号识别**
   - 格式：`[YYYY]省[城市]民初/行初/刑初[序号]号`
   - 无案号时：使用`{原告}诉{被告}{案由}`

2. **当事人信息**
   - 原告姓名/名称
   - 被告姓名/名称
   - 第三人（如有）

3. **案由识别**
   - 从诉讼请求或争议内容中提取

4. **其他关键信息**
   - 法院名称
   - 立案日期
   - 文书日期

### 零幻觉原则

- **绝对禁止**猜测或推断信息
- 无法确认的信息必须标注"待确认"
- 每个信息都要注明来源文档
- 多文档交叉验证

## 质量标准

| 指标 | 要求 |
|------|------|
| 分析速度 | < 3分钟/文档 |
| 整体准确率 | > 90% |
| OCR识别准确率 | > 95% |
| 关键字段准确率 | 100% |

## 案号识别规则

### 标准案号格式
```
[YYYY]省[城市]民初/行初/刑初[序号]号
```
例如：`[2025]京0105民初1234号`

### 无案号处理
1. 从文档中提取当事人信息
2. 生成格式：`{原告}诉{被告}{案由}`
3. 示例：`甲诉乙著作权侵权案`

### 关键要点
- **严禁捏造案号**
- 直接使用当事人信息
- 案由需准确识别
- 目录直接放在output下（禁止cases子文件夹）
